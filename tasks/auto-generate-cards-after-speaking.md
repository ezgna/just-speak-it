# 録音停止後に英語カード生成まで自動で走らせる

## やること

通常の音声入力フローでは、ユーザーが `Speak it` を押して話し、停止ボタンを押したら、文字起こし、カード分割、英訳カード生成、保存、表示更新までを自動で進める。

`Split it` や `Make cards` のような段階ごとの実行ボタンを、通常フローでは必須にしない。

フラッシュカードは画面上にずっと表示され続け、新しい録音から生成されたカードは準備できたら自然に追加・更新される状態を目指す。

## 背景

現在の体験は、録音後にユーザーが `Split it`、`Make cards` のような次の操作を選ぶ前提になっている。

ただ、このアプリの中心体験は「話したら英語カードができて、すぐ復習できる」ことなので、生成工程をユーザーに意識させすぎると重く感じる。

基本は録音開始と停止だけにして、生成工程は裏側で進める方が自然そう。

一方で、音声認識の日本語文字起こしが間違うことはあるため、後から文字起こしを修正して、そこから英語カードを再生成できる逃げ道は残す。

## 受け入れ条件

- 通常の音声入力では、ユーザー操作は `Speak it` を押すことと停止することを基本にする。
- 停止後に、文字起こし、整形、カード分割、英訳カード生成、保存までが自動で始まる。
- 通常フローで `Split it` や `Make cards` を押さないと先へ進めない状態にしない。
- 既存のフラッシュカード表示は、録音中・生成中も消えずに残る。
- 新しいカードは生成完了後に、既存のフラッシュカード表示へ自然に反映される。
- 生成中は、必要最小限の進行状態だけを見せる。ユーザーに次の工程を選ばせない。
- 文字起こしや生成に失敗した場合は、最後の録音または文字起こしから再試行できる。
- 日本語の文字起こしが間違っていた場合、後から本文を修正できる。
- 修正した日本語本文から、カード分割と英訳カード生成を再実行できる。
- 再生成時に、同じ録音由来の古いカードが重複して残らない。
- 再生成で既存カードを置き換える場合、学習状態や timestamp の扱いを破綻させない。

## 実装メモ

- 既存候補:
  - `src/app/(tabs)/index.tsx`
  - `src/components/generated-practice-preview.tsx`
  - `src/components/slack-flashcard-lab.tsx`
  - `src/lib/backend/transcription.ts`
  - `src/lib/backend/practice.ts`
  - `supabase/functions/transcribe/index.ts`
  - `supabase/functions/prepare-practice-draft/index.ts`
  - `supabase/functions/complete-practice/index.ts`
- 現在の `preparePracticeDraft` と `completePracticeDraft` の段階は、内部工程としては残してよい。
- UI 上では、段階ごとのボタンを前面に出さず、録音停止後に内部で順番に呼ぶ形へ寄せる。
- 文字起こし修正後の再生成は、少なくとも「修正済み本文から draft を作り直す」「その draft からカードを作り直す」までを一連で走らせたい。
- 再生成の対象は、`diaryEntry` または `practiceGeneration` 単位で明確にする。
- 最小実装では、まず音声入力の happy path だけを自動化し、詳細編集 UI は後続に分けてもよい。
