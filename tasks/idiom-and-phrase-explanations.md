# イディオム・熟語をいい感じに解説する

## やること

英訳カードに出てくるイディオム、熟語、自然な言い回しを、タップしてわかりやすく解説できるようにする。

ただの辞書説明ではなく、「なぜこの英語になるのか」「どういう場面で使うのか」「日本語の感覚とどこが違うのか」まで、学習者が腹落ちしやすい形で見せる。

## 背景

現在の英訳カードは、日本語と英語の対応を見ることはできるが、英語表現の中にあるチャンク、熟語、ニュアンスまでは説明していない。

例として `on my way home` や `curious about` のようなまとまりを、単語ごとの直訳ではなく、使い方ごと覚えられるようにしたい。

## 受け入れ条件

- 英訳カード内の重要な表現が、イディオム・熟語・自然な言い回しとして抽出される。
- 解説対象は多すぎず、1カードあたり本当に覚える価値がある表現に絞る。
- 各解説には、少なくとも次を含める。
  - 表現そのもの
  - やさしい日本語の意味
  - このカード内でのニュアンス
  - 似た場面で使える短い例文
  - 直訳しにくい場合の注意点
- 解説は英語学習者向けに自然な日本語で書く。
- 説明文が長すぎてカード体験を邪魔しない。
- 復習中は、必要なときだけ解説を開ける。
- 解説を見ても、既存のスワイプ復習・音声再生・学習状態保存の操作を邪魔しない。
- 生成失敗や解説なしの場合でも、カード生成自体は壊れない。

## 実装メモ

- 既存候補:
  - `src/components/slack-flashcard-lab.tsx`
  - `src/components/english-practice-deck.tsx`
  - `src/lib/backend/practice.ts`
  - `supabase/functions/complete-practice/index.ts`
  - `supabase/migrations/20260531000000_init_word_timestamp_schema.sql`
  - `src/lib/supabase/database.types.ts`
- `complete-practice` で英訳を作るタイミングに、表現解説も構造化して返すのが自然そう。
- DB に保存するなら、`translation_cards` へ JSON カラムを足すか、`translation_card_phrase_notes` のような別テーブルに分ける案がある。
- UI は、英語文中のチップ、下線、またはカード下部の小さな「表現」ボタンから開く形が候補。
- デザインラボには phrase chip 的な表示があるため、見た目の方向性の参考にできる。
- 最初は「カード生成時に静的解説を保存する」方が、復習中に追加 API 呼び出しをしなくて済む。
