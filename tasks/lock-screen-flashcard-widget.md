# ロック画面やウィジェットでフラッシュカードを表示する

## やること

Remindify 的に、ロック画面やホーム画面ウィジェットで復習用フラッシュカードを表示できるようにする。

アプリを開かなくても、今日覚えたい英訳カードが自然に目に入り、短い空き時間に復習できる状態を目指す。

あわせて、ロック画面やウィジェット上で録音処理までできるかを調査する。録音が直接できない場合は、アプリ起動、Live Activity、App Intent、通知アクションなど、現実的な代替導線を検討する。

## 背景

現在のフラッシュカード復習はアプリ内の画面に入ってから行う体験になっている。

ただ、英語表現の暗記は「何度も目に入る」ことが効きやすいため、Remindify のようにロック画面やウィジェットへカードを出せると、復習の接触回数を増やせる。

さらに、このアプリでは話す練習も重要なので、表示だけでなく、その場で音読・録音・発話チェックへ進める余地があるかを確認したい。

## 受け入れ条件

- ロック画面またはホーム画面ウィジェットで、復習対象のフラッシュカードを表示できる。
- 表示するカードは、今日の復習対象や未習得カードなど、ユーザーにとって意味のある候補から選ばれる。
- 日本語面、英語面、または短いヒントのどれを表示するかを設計する。
- ウィジェット上の表示は、長文カードでも破綻しない。
- ウィジェットをタップしたとき、該当カードの復習画面または今日タブへ自然に遷移できる。
- カード内容が古いまま残り続けないよう、更新頻度とキャッシュの扱いを決める。
- ロック画面やウィジェット上で録音開始・録音保存・音声権限要求が可能かを調査する。
- 録音が直接できない場合は、最短で録音画面へ遷移する代替案をまとめる。
- iOS の WidgetKit、Live Activities、App Intents、通知アクションの制約を確認し、Expo/React Native で実現する場合の実装方針を決める。
- 実装前に、Expo の該当バージョンのドキュメントと iOS 側の最新制約を確認する。

## 実装メモ

- 既存候補:
  - `src/components/slack-flashcard-lab.tsx`
  - `src/hooks/use-translation-card-groups.ts`
  - `src/lib/backend/practice.ts`
  - `src/app/(tabs)/index.tsx`
  - `ios/`
- まずは録音込みで作り始めるのではなく、iOS の制約調査を先に行う。
- 通常の WidgetKit ウィジェットは表示とタップ導線が中心になる可能性が高いため、録音はアプリ内へ deep link する形が現実的かもしれない。
- Live Activity や通知アクションで「今のカードを開く」「録音画面を開く」導線を作れるかを確認する。
- 復習対象カードの選定ロジックは、アプリ内の復習キューと揃える。
- ウィジェット用に必要な最小データだけを native 側から読める場所へ同期する必要があるか確認する。

## 未決

- 最初に対応するのはロック画面ウィジェット、ホーム画面ウィジェット、Live Activity のどれにするか。
- 表示する面を日本語にするか、英語にするか、時間帯や復習状態で切り替えるか。
- ウィジェット上でカードをめくる体験まで作るか、タップしてアプリで復習するだけにするか。
- 録音できない場合の代替導線を、今日タブにするか、該当カードの録音画面にするか。
- Android でも同等のウィジェット体験を作るか。
